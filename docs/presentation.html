What We Built & Why

The Problem

3D games and films rely on skeletal animation — deforming a mesh by blending transforms from a hierarchy of bones. Implementing this from scratch requires almost every topic in a graphics course.

Our Solution

  • Load glTF 2.0 skinned characters
  • Sample & interpolate keyframe animations
  • Upload bone matrices to the GPU (UBO)
  • Deform the mesh in the vertex shader
  • Interactive camera, playback, blending

Why glTF?

Open standard by Khronos (same group behind OpenGL). Stores geometry, skeleton, skin, animations, and textures in one file. The "JPEG of 3D".

Vertex Shader — GPU Skinning

Each vertex is influenced by up to 4 bones. The shader computes a weighted sum of the 4 bone matrices and transforms the vertex position and normal through it.

v' = w₀·M[b₀]·v + w₁·M[b₁]·v
    + w₂·M[b₂]·v + w₃·M[b₃]·v

Fragment Shader — Shading

Implements Blinn-Phong illumination: ambient + diffuse (NdotL) + specular (half-vector). Samples the character's albedo texture. Normals are transformed with the inverse-transpose for correctness.

Geometric Objects & Transformations

Bone Hierarchy = Scene Graph

Each bone has a parent index and a local TRS transform (Translation, Rotation, Scale). A single forward pass computes global world transforms:

global[i] = global[parent[i]] × local[i]

Skinning Matrix

Transforms a vertex from its bind-pose position to its animated world position:

M_skin[i] = global[i] × invBind[i]

The invBind matrix moves vertices out of world space back into bone-local space before animating.

Animation Sampling & Interpolation

Keyframe Channels

Each animation stores keyframe tracks per bone: separate time arrays and value arrays for T (position), R (rotation), S (scale). On each frame, we binary-search for the surrounding keyframes and interpolate.

Rotation — SLERP

Rotations are stored as unit quaternions and interpolated with Spherical Linear Interpolation — takes the shortest arc on the rotation sphere. No gimbal lock.

R = slerp(R_prev, R_next, α)

Translation & Scale — LERP

Positions and scales are interpolated linearly between keyframes. α = (t − t₀) / (t₁ − t₀).

Animation Blending (Week 6)

To crossfade between two clips, we sample both at TRS level simultaneously and blend:

  • Source clip: time frozen at blend start
  • Target clip: time advancing from 0
  • Blend α goes from 0 → 1 over 0.4 s
  • SLERP on rotations prevents stretching

Why Not Matrix LERP?

Linearly interpolating two rotation matrices does not stay orthogonal — the result shears and scales the mesh. TRS-level SLERP is the correct solution used in production.

Buffers & Texture Mapping

Vertex Buffer Object (VBO)

All per-vertex data is packed into a single interleaved buffer:

attrib 0 — position (vec3)
attrib 1 — normal (vec3)
attrib 2 — UV (vec2)
attrib 3 — bone IDs (ivec4)
attrib 4 — weights (vec4)

Uniform Buffer Object (UBO)

128 skinning matrices (mat4) uploaded as a std140 UBO — a single GPU buffer that all vertex shader invocations read simultaneously. 8 KB per frame update.

Albedo Texture

Loaded from glTF via stb_image, uploaded to a 2D GL texture with mipmaps. The fragment shader samples it with the vertex's UV coordinates. If no texture exists, a neutral grey fallback is used.

Depth Buffer

Standard GL_DEPTH_TEST ensures correct occlusion. Intentionally disabled when drawing the skeleton overlay so bone lines are always visible through the mesh surface.

Viewing & Projection · Input & Interaction

Orbit Camera

Parameterised by yaw, pitch, radius, target. The view matrix is reconstructed from spherical coordinates each frame. Target defaults to hip height so the character is centred.

Perspective Projection

45° vertical FOV, aspect ratio updated on window resize. Near plane 0.05 m, far plane 500 m — avoids z-fighting with the ground grid while keeping precision near the character.

MVP Transform

gl_Position = Proj × View × Model × skinnedPos

Model = rootTransform (handles glTF axis conventions)

Input System

  • LMB drag → orbit (yaw & pitch)
  • RMB drag → pan (translate target)
  • Scroll → zoom (radius)
  • Space → play / pause
  • N → smooth clip crossfade
  • J / L → frame-step scrubbing

Dear ImGui Panel

Provides a GUI overlay with sliders, buttons, and a clip selector — all without blocking the GLFW event loop. ImGui callbacks chain our existing GLFW callbacks automatically.

Shading

Blinn-Phong Illumination

One directional light. Three components:

  • Ambient (0.15) — prevents fully black shadows
  • Diffuse (0.8) — Lambert: max(N·L, 0)
  • Specular (0.2) — half-vector: (N·H)³²

Normal Transformation

When a non-uniform scale is present, normals cannot be transformed with the same matrix as positions. The correct transform is the inverse-transpose:

n' = mat3(transpose(inverse(M_skin))) × n

Skinned Normal Pipeline

The normal must be skinned with the same weighted bone matrices as the position, then re-normalised after the inverse-transpose. If you skip this, shading breaks when bones scale non-uniformly.

Ground Grid

A flat line grid rendered with a custom shader. The fragment shader fades lines to zero alpha based on distance from the camera, using smoothstep — demonstrates simple procedural shading.

Challenges We Solved

🌀
Mesh "tornado" / model tiny & upside-down
Non-joint ancestor nodes (Z_UP, Armature) in glTF carry orientation. Fixed by DFS-accumulating transforms of non-joint nodes into a rootTransform matrix applied as uModel.
⌨️
Speed keys not working on macOS
GLFW key codes differ by layout. Used glfwSetCharCallback which fires Unicode codepoints ('+'=43, '-'=45) regardless of keyboard layout.
🦴
Skeleton overlay invisible (inside mesh)
Depth test culled the debug lines since they sit exactly at bone positions inside geometry. Fixed by disabling GL_DEPTH_TEST for the bone draw pass and restoring it after.
📦
layout(binding=0) compile error on macOS GL 4.1
OpenGL 4.1 does not support explicit UBO binding qualifiers in GLSL. Removed from shader and used glUniformBlockBinding() at load time instead.
🔀
Animation blending caused mesh shearing
Matrix-level lerp between animation poses is not rotation-correct. Solution: blend at TRS level — SLERP quaternion rotations, LERP translation and scale before building the mat4.

Bottleneck Analysis

At this model size (~7 600 triangles) the GPU is entirely idle most of the frame. The limiting factor is vsync, not computation. On a production character (~100 k triangles, 200 bones) the UBO and skinning shader would still scale well.

Scalability Notes

  • 128-bone UBO limit → fine for humanoids
  • SSBO on GL 4.3 would allow unlimited bones
  • Animation blending adds one extra TRS sample pass — negligible
  • ImGui overhead: ~0.1 ms per frame

What's Next — Possible Extensions

🌟

Multiple Blend Layers

Layer upper-body and lower-body animations independently (additive blending). Standard in game engines for playing a "reload" animation on the arms while the legs continue a walk cycle.

☀️

Shadow Mapping

Render the scene from the light's perspective to produce a depth map, then test each fragment against it in the main pass. Connects directly to the Ray Tracing syllabus topic — shadow rays vs. shadow maps are the key trade-off.

🦾

Inverse Kinematics (IK)

Procedurally compute joint angles so a hand or foot reaches a target position. Extends the bone hierarchy system with an iterative solver (CCD or FABRIK).

🪨

PBR Materials

glTF 2.0 natively supports metallic-roughness PBR. Replace Blinn-Phong with a physically-based BRDF (Cook-Torrance) and add an environment cubemap for image-based lighting.

Conclusion

What This Project Covers

Almost every topic on the syllabus appears as a concrete, working implementation — not just theory. The skinning pipeline connects graphics systems, transformations, shaders, buffers, and shading into a single coherent system.

Key Takeaways

  • Bone-array order is the #1 source of skinning bugs
  • Blending must happen at TRS level, not matrix level
  • macOS GL 4.1 has real constraints to design around
  • A debug overlay (skeleton lines) is essential — isolates animation bugs from skinning bugs
