Core Equation

Everything in the project serves this one equation:

boneMatrix[i] = global[i] × invBind[i]

invBind moves a vertex from world space back to bone-local space. global then re-places it in its animated world position.

Key Insight — Bind Pose vs. Projection

What the Animator Class Does

  • Advances currentTime each frame using fmod(t + dt, duration)
  • Samples T · R · S keyframes for every bone channel
  • Builds local[] → global[] → skinning[] matrices
  • Uploads skinning[] to the GPU via UBO
  • Supports smooth crossfade blending between two clips

Steps 1–5 · Keyframe Sampling & Local Transform

Step 1 — Channel Data (from glTF)

struct Channel {
  int boneIdx;
  vector<float> times; // [0.0, 0.5, 1.0 …]
  vector<vec3> T; // translation keys
  vector<quat> R; // rotation keys
  vector<vec3> S; // scale keys
};

One channel = one bone. Keys are sampled at runtime.

Step 2 — Advance Time

time += dt × speed;
time = fmod(time, duration);

Step 3 — Find Surrounding Keyframes

Binary-search times[] to find t_i ≤ currentTime ≤ t_{i+1}, then compute α = (t − t_i) / (t_{i+1} − t_i).

Step 4 — Interpolation

// Translation & Scale — LERP
T = mix(T_i, T_{i+1}, α);
S = mix(S_i, S_{i+1}, α);

// Rotation — SLERP (shortest arc on unit sphere)
R = slerp(R_i, R_{i+1}, α);

SLERP is essential — linear quaternion interp skips the correct arc and causes jerky rotation.

Step 5 — Build Local mat4

local[i] = translate(T)
× mat4_cast(R)
× scale(S);

Position of this bone relative to its parent. Bones with no channel use their bind-pose transform.

Steps 6–8 · Hierarchy, Bone Matrices & GPU Upload

Step 6 — Hierarchy Forward Pass ⭐ MOST IMPORTANT

// Single forward loop — parent always before child
for (int i = 0; i < n; i++) {
  int p = skeleton[i].parent;
  global[i] = (p < 0)
    ? local[i] // root bone
    : global[p] * local[i]; // child inherits parent
}

Shoulder rotates → Arm must follow → Hand must follow. Skipping this makes every bone move independently.

Common Bugs Without Correct Hierarchy

  • Skeleton looks completely wrong
  • Arms detach from shoulders
  • Character explodes on movement
  • Fix: ensure parent index is correct and loop is ordered parent-first

Step 7 — Bone Matrix = Global × InvBind

for (int i = 0; i < n; i++)
  skinning[i] = global[i] * invBind[i];

invBind undoes the bind-pose world transform. At t=0 the result is identity — our validation test (press 0).

Step 8 — Upload to GPU (UBO)

glBufferSubData(GL_UNIFORM_BUFFER,
  0, n * sizeof(mat4), skinning.data());

128 mat4s in a std140 Uniform Buffer Object. The vertex shader reads them by bone index. One call per frame — 8 KB for 128 bones.

Step 9 · GPU Skinning — Numerical Example

Setup — One Vertex, Two Bones

Original Vertex
v = (1, 0, 0)
Bone 0 — weight 0.7
Rotate 90° around Z
Bone 1 — weight 0.3
Translate +2 in X

Apply Each Bone Matrix

M₀ × v = (0, 1, 0) // rotated 90° → Y axis
M₁ × v = (3, 0, 0) // translated +2 in X

Blend by Weights ← This IS Skinning

v' = 0.7 × (0,1,0) + 0.3 × (3,0,0)
   = (0, 0.7, 0) + (0.9, 0, 0)
   = (0.9, 0.7, 0) ← final position

This Is Exactly Your Vertex Shader

mat4 skin =
  weights.x * bones[boneIDs.x] +
  weights.y * bones[boneIDs.y] +
  weights.z * bones[boneIDs.z] +
  weights.w * bones[boneIDs.w];
gl_Position = Proj*View*Model * skin * vec4(v,1);

Why This Makes Smooth Animation

  • Vertex is NOT fully rotated or fully translated
  • It is a weighted blend of both
  • Nearby vertices share similar weights → smooth deformation at joints
  • This is called Linear Blend Skinning (LBS)

VAO / VBO Layout

0 position vec3
1 normal vec3
2 uv vec2
3 boneIDs ivec4
4 weights vec4

Single interleaved VBO per mesh. Bone data lives alongside geometry.

Blinn-Phong Shading

  • Ambient 0.15 — no pure black
  • Diffuse 0.8 — max(N·L, 0)
  • Specular 0.2 — (N·H)³²
  • Normal via inverse-transpose of skin matrix
  • Albedo from glTF texture (stb_image)

Depth Buffer

Standard GL_DEPTH_TEST for occlusion. Disabled for bone overlay so lines show through the mesh. Re-enabled after.

Inverse-Transpose Normal

n' = mat3(transpose(
  inverse(skin))) × n

Required when bones have non-uniform scale, otherwise normals break.

Orbit Camera

Parameterised by yaw, pitch, radius, target. View matrix rebuilt from spherical coordinates each frame. 45° FOV perspective, aspect updated on resize.

gl_Position = Proj × View × rootTransform × skinnedPos

Key Challenges & Solutions

🌀
Model tiny & upside-down
Non-joint ancestor nodes (Z_UP, Armature) carry orientation. Fixed: DFS walk accumulates their transforms into rootTransform, applied as uModel.
📦
layout(binding=0) compile error on macOS GL 4.1
GL 4.1 doesn't support explicit UBO binding qualifiers in GLSL. Removed from shader — binding set via glUniformBlockBinding() at load time instead.
🦴
Skeleton overlay invisible (inside mesh)
Depth test culled lines sitting inside geometry. Fixed: glDisable(GL_DEPTH_TEST) for the bone draw pass, restored immediately after.
🔀
Animation blending caused mesh shearing
Matrix-level lerp breaks rotation orthogonality. Solution: blend at TRS level — SLERP quaternion rotations, LERP translation and scale before building mat4.
⌨️
Speed keys broken on macOS keyboard layout
GLFW key codes differ by layout. Used glfwSetCharCallback — fires actual Unicode codepoints (+/−), works on any keyboard layout.

  • TRS-level animation blending with SLERP
  • Dear ImGui interactive debug panel

Key Takeaways

  • Core equation: v' = Σ weight × boneMatrix × v
  • Hierarchy is the most critical step — skip it and the skeleton breaks
  • Blending must be at TRS level, not matrix level
  • Bind pose ≠ projection — completely separate systems
  • macOS GL 4.1 has real constraints (no SSBO, no UBO binding qualifier)

Future Extensions
